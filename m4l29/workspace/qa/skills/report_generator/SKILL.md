## 生成步骤

### 第一步：收集数据

在沙盒中读取输入文件：
```bash
cat /workspace/defect_list.json 2>/dev/null || echo '{"defects":[]}'
```

### 第二步：按模板生成报告

根据模板填充全部字段。必须做到：
- **不捏造数据**：所有数字来自实际测试结果和缺陷清单
- **结论有依据**：不通过一定有 P0 缺陷或核心流程失败作为支撑
- **风险可追溯**：每个风险项关联到具体的缺陷 ID 或用例 ID

### 第三步：写入报告

使用 `sandbox_file_operations(action="write")` 将完整报告写入：
```
sandbox_file_operations(
  action="write",
  path="/workspace/test_report.md",
  content="（完整 Markdown 报告内容）"
)
```

### 第四步：read-back 验证

```
sandbox_file_operations(
  action="read",
  path="/workspace/test_report.md"
)
```

确认文件字节数与写入内容长度匹配。不匹配 → 重试写入。

## 特殊情况处理

| 场景 | 处理方式 |
|------|---------|
| 零缺陷 | 缺陷统计表格写"无"，风险评估写"未发现高风险项" |
| 无测试结果 | errcode=1, errmsg="请先执行 test_executor 生成测试结果" |
| 缺陷清单 JSON 损坏 | errcode=1, errmsg="defect_list.json 格式错误，请检查" |
| 部分模块未测试 | 对应模块表格写"未测试"，不计入通过率 |
